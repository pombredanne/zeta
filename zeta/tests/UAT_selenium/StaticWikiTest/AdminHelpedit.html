== Administration

=== Site Admin

The site administrator has access to all the resources. Following tabs can be found
in the '/ site-admin '/ page.

==== Site info

Site info tab contains the global configuration fields which are applicable
to all projects. These fields are read-only and can be modified
only in the .ini file.

==== User management

'/ Users '/ tab is used for assigning site-level permissions to registered users.
These permissions do not cover project level permissions (project-level permissions
are managed in the respective project's admin tab)
Site level resources include license create, view, email id view etc..
Enabling or Disabling users do not have any effect now. This feature is
left for future development.

==== Projects

Enabling or Disabling projects do not have any effect now. This feature is
left for future development.

==== Permissions

'/ Permissions '/ tab is used for creating and updating permission groups.
A set of permission names can be grouped under a permission group. Know
more about [[ /help/pms | Permission Managment System ]]


==== License

'/ licensetab '/ gives a snapshot of all the license available in the site. License pages can be
accessed only through this page and project's admin page.

=== Project Administrator

Whoever creates a project becomes the project's administrator. Subsequently any registered user
can be re-assigned as the project's administrator by //site-admin// or by the project's current
administrator. The project admin has unrestrictedaccess to all pages related to the project.

'' Being a //project administrator// is to administor project meta details. For contributing to
a project join one (or more) of project's team.''

The project meta details are classified as,

: Project :: \
Project admin can give an overview description of the project and upload the project logo and icon which \
will be displayed in all project-pages.

:Component ::\
A project can be divided into components for the purpose of managing the same. \
Each component is assigned an //owner//.


:Milestone ::\
The project admin can create milestones for the project and specify a //due date//. Milestones can be closed by \
completing or cancelling it. Milestones can be used to track key dates in the project.

:Versions ::\
The project admin can create and remove versions which can be used to track releases.

> '' Tickets can be associated to project components, milestones and versions. ''

:Teams ::\
Users can be added or removed from project teams. Team names can be created only by site admininistrator \
and are common to all projects. Every user who participates in the project needs to be assigned to a team, \
but those with multiple roles can be associated with more than one team.

:Permissions ::\
Permission management is done at a team level by adding or removing permissions to teams.
Know more about [[ /help/pms | Permission Managment System ]]
